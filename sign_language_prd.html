Sign Language Gesture Recognition — PRD
Version 1.0 Status: Draft Phase: Option A (Rule-Based) Owner: Karthik
Overview
Problem statement
Users with hearing or speech impairments have no gesture-based interaction method for web apps. This project enables real-time sign language gesture recognition via webcam to trigger UI actions (like, confirm, reject, etc.) — making the web more accessible and interactive.
Solution
A browser-based app using MediaPipe Hands to detect hand landmarks in real-time, classify gestures using rule-based logic, and map them to UI actions — no backend, no model training required in Phase 1.
